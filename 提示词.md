项目概述

请开发一个名为“蓝奏云分片助手”的带图形界面（GUI）的桌面应用。其核心目标是：允许用户通过简单的点击操作，将任意大文件或文件夹安全地拆分、加密为一系列可通过蓝奏云直接上传的小文件；同时，该应用也提供一个独立的还原程序，让接收方能够通过图形界面轻松地将这些小文件还原并解密为原始内容。整个流程需对终端用户完全隐藏命令行操作。

详细功能与设计要求

第一部分：图形化分片工具 (split_gui.exe)
• 核心界面：一个简洁的窗口，包含“选择文件/文件夹”按钮、进度条、操作日志文本框和“开始处理”按钮。

• 处理流程（用户点击后自动完成）：

    1.  输入与压缩：用户选择目标后，若为文件夹，则应用自动在后台将其打包为ZIP。
    2.  加密：必须对上述ZIP包进行AES-256加密。密码不得硬编码在源码中，而应通过弹窗或界面输入框由用户临时设置并确认。在代码层面，需对用户输入的密码进行加盐哈希处理后再作为加密密钥，以增加暴力破解难度。
    3.  分片与混淆：将加密后的文件按49MB（大小可配置）分片。为每个分片生成一个随机文件名（如f5a2e8.pkg），其扩展名必须从蓝奏云允许的后缀列表中选取（例如使用 .pkg, .cpk, .txt 等，避免使用非法后缀如.bin）。
    4.  生成清单：创建一个名为manifest.dat的文件（可使用.txt, .conf等允许后缀），以JSON格式记录所有分片的元数据（顺序、混淆名、大小、SHA256）以及源文件信息。绝对禁止在其中存储明文密码。
• 输出：在用户指定位置生成一个文件夹，内含所有混淆命名的分片文件、清单文件以及一个独立的restore_gui.exe还原程序。

第二部分：图形化还原工具 (restore_gui.exe)
• 核心界面：一个独立的窗口程序，界面包含“选择清单文件”或“自动识别”按钮、进度条和日志框。

• 自动还原流程：

    1.  用户将所有分片文件和manifest.dat、restore_gui.exe放在同一目录下。
    2.  运行restore_gui.exe，点击“开始还原”按钮。
    3.  程序自动查找并验证manifest.dat及所有分片的完整性与一致性。
    4.  验证通过后，弹窗提示用户输入解密密码。
    5.  使用用户输入的密码尝试解密并重组出原始的ZIP文件。
    6.  最后，提供选项让用户选择解密后ZIP的保存位置，并可选自动解压。
• 错误处理：所有步骤都应有明确的图形化提示（成功、失败、密码错误等）。

第三部分：技术实现规范
• 语言与框架：使用 Python 3.13.\*，GUI框架推荐使用 PySimpleGUI 或 Tkinter 以保证简洁和打包体积。

• 依赖管理：使用 Poetry 管理项目依赖（如pycryptodome用于加密，pyinstaller用于打包）。

• 打包发布：将分片工具和还原工具分别打包为独立的、无需Python环境的Windows可执行文件（split_gui.exe 和 restore_gui.exe）。

第四部分：文档与用户体验
• 提供一份图文并茂的README.md或使用指南.pdf。文档必须完全基于图形界面操作进行讲解，例如：

    ◦ 第一步：运行split_gui.exe，点击“选择文件”。

    ◦ 第二步：在弹出的窗口中设置并牢记你的加密密码。

    ◦ 第三步：点击“开始”，等待完成。

    ◦ 第四步：将生成的整个文件夹上传至蓝奏云。

    ◦ 第五步：下载方下载整个文件夹后，直接运行里面的restore_gui.exe，点击“还原”并输入密码即可。

• 整个设计需围绕“零命令行知识用户”的使用场景展开，确保流程直观、安全、可靠。
